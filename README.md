Alumna

Chiara Diaz

## Descripción

Este proyecto propone la automatización de consultas repetitivas del área de soporte técnico mediante la integración de Jira Service Management, Confluence y un agente virtual basado en inteligencia artificial.

## Objetivo

Reducir la carga operativa del soporte técnico nivel 1 automatizando la entrega de instructivos y documentación existente en Confluence.

## Herramientas Utilizadas

* Jira Service Management
* Confluence
* Jira Rovo
* BPMN 2.0
* GitHub

## Diagramas

* AS-IS: proceso actual de atención de tickets.
* TO-BE: proceso automatizado mediante chatbot.

## Gestión de Estados

Estados principales:

* ST_ESPERANDO_TICKET
* ST_ANALIZANDO_CONSULTA
* ST_BUSCANDO_CONFLUENCE
* ST_VALIDANDO_RESPUESTA
* ST_ESCALADO_HUMANO
* ST_RESUELTA

## Beneficios Esperados

* Menor carga operativa.
* Respuesta inmediata a consultas frecuentes.
* Mejor aprovechamiento de la base de conocimientos.
* Escalamiento únicamente cuando sea necesario.
